Getting Started with Sphinx

A beginner-friendly guide to help you create documentation using Sphinx. This guide covers everything from installation to building your first HTML documentation site.

Prerequisites​

Make sure you have the following before getting started:

  • Python 3.7 or higher
  • pip (Python package manager)
  • A terminal or command prompt
  • A code editor like VS Code or Sublime Text
  • (Optional but recommended) Git

Step 1: Install Sphinx and a Theme​

Install Sphinx and the Read the Docs theme:

pip install -U sphinx sphinx_rtd_theme

If you plan to use Markdown files instead of reStructuredText:

pip install myst-parser

Step 2: Set Up Your Project Structure​

Create the following folder structure:

my-project/
├── docs/             # Documentation folder
└── src/              # Your project code (optional)

Navigate into the docs folder:

mkdir my-project
cd my-project
mkdir docs
cd docs

Step 3: Run Sphinx Quickstart​

Generate the initial documentation structure:

sphinx-quickstart

Answer the prompts. Recommended options:

  • Separate source and build directories: yes
  • Project name, author: Enter appropriate values
  • Project release/version: Optional
  • Create Makefile: yes
  • Create Windows command file: yes

This will generate these important files:

  • conf.py – Sphinx configuration
  • index.rst – Main documentation page
  • Makefile / make.bat – For building output

Step 4: Configure Sphinx​

Edit conf.py in the docs folder:

  1. Add your project path:

    import os
    import sys
    sys.path.insert(0, os.path.abspath('../src'))

  2. Enable useful extensions:

    extensions = [
        'sphinx.ext.autodoc',
        'sphinx.ext.viewcode',
        'sphinx.ext.napoleon',  # Google/Numpy-style docstrings
        'myst_parser',          # for Markdown support (optional)
    ]

  3. Set the HTML theme:

    html_theme = 'sphinx_rtd_theme'

Step 5: Add Your Documentation Content​

Update index.rst to include a table of contents:

Welcome to My Project’s Documentation!
======================================

.. toctree::
   :maxdepth: 2
   :caption: Contents:

   introduction
   usage

Create new files like introduction.rst and usage.rst in the same folder and add your content there.

Example introduction.rst:

Introduction
============

This project is a simple example to demonstrate Sphinx setup.

Step 6: Generate HTML Output​

From the docs folder, run:

make html

The generated site will be available at:

_build/html/index.html

Open this file in your browser to preview the site.

Step 7: Live Preview (Optional)​

Install sphinx-autobuild for live-reloading:

pip install sphinx-autobuild

Run:

sphinx-autobuild . _build/html

Your browser will reload automatically whenever you save changes.

Step 8: Add API Documentation (Optional)​

To document your Python code automatically:

  1. Add docstrings to your code in the src/ folder

  2. Run:

    sphinx-apidoc -o source/ ../src/

  3. This generates .rst files for your modules. Include them in index.rst.

Step 9: Publish Your Docs​

You can host your documentation using:

  • Read the Docs: Connect your repo and add a .readthedocs.yaml file
  • GitHub Pages: Push the _build/html folder to the gh-pages branch
  • Netlify / Vercel: Deploy using the docs/_build/html folder as static output

Tips​

  • Use napoleon extension for clean docstring formatting
  • Use viewcode to let users browse your code from docs
  • Use myst_parser if you prefer writing docs in Markdown
  • Avoid editing _build/ manually – it's auto-generated

What Next?​

You’ve set up your first documentation site using Sphinx. As your project grows, you can include diagrams, search, PDF exports, and more.

  • Try PDF output: make latexpdf
  • Use themes like sphinx-rtd-theme
  • Add versioning via sphinx-multiversion
